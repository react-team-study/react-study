리액트는 오직 V(view)만 신경쓰는 라이브러리 이다.

초기렌더링
- 리액트는 초기 랜더링을 다루는 render 함수가 있다.
- render 함수는 컴포넌트가 어떻게 생겼는지 정의하는 역할을 한다.
- 이 함수는 html 형식의 문자열을 반환하지 않고, 뷰가 어떻게 생겼고 어떻게 동작하는지에 대한 정보를 지닌 객체를 반환한다.

조화과정
- 컴포넌트에서 데이터의 변화가 있을때 변화에 따라 뷰가 변형되지 않고, render 함수를 통하여 새로운 요소로 갈아 끼운다. 
- 가상돔을 통해 이전 DOM과 새로운 DOM 트리의 차이점을 비교하여 최소한의 연산으로 DOM 트리를 업데이트 한다.


리액트 특징
- Virtual DOM을 사용한다.
- Virtual DOM을 사용하여 실제 DOM에 접근할여 조작하는 대신, 이를 추상화한 자바스크립트 객체를 사용한다.
- 리액트에서 데이터가 변하여 웹 브라우저에 실제 DOM을 업데이트 할때는 다음 세가지 절차를 밟는다.
1. 데이터를 업데이트 하면 전체 UI를 Virtual DOM에 리렌더링 한다.
2. 이전 Virtual DOM에 있던 내용과 현재 내용을 비교한다.
3. 바뀐부분만 실제 DOM에 적용한다.

작업환경 설정
1. npm || yan 설치
2. 바벨, 웹팩 설치 
 * 바벨이란 ? 이크마에서 2015년도에 정의한 ES6 문법을 웹브라우저에서 호환될수 있도록 하는 도구
 * 웹팩이란 ? 모듈화된 코드를 한 파일로 합치고 코드를 수정할 때마다 웹브라우저를 리로딩하는 도구
